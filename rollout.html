1. Platform Overview & Roles

CleanSpace Stay manages all workforce travel — booking requests, confirmations, spend tracking, and QuickBooks reconciliation — in one platform with a companion mobile app.

2. Rollout Steps

Day 1 — Admin Setup

1

Create Projects

Add all active projects with project codes, locations, and dates. These are what workers select when requesting travel.

2

Add Project Managers

Create PM accounts (email + password login). Assign each PM to their projects.

3

Add Foremen

Create Foreman accounts (phone number login). Assign to projects. Foremen will be the ones who introduce the app to their crews.

4

Connect QuickBooks

Go to Settings → Integrations → Connect QuickBooks. Follow the OAuth flow. Map each project to its QB Customer/Class. (See Section 4 below.)

5

Set Up Email Integration

Either connect Microsoft Graph (preferred) or set up auto-forwarding from travel@ to booking@cleanspacestay.com. (See Section 3 below.)

Day 2-3 — PM & Foreman Onboarding

6

Walk PMs through the platform

15-minute screen share: show them the dashboard, how to approve requests, and where to see spend. Share the Help Center link.

7

Walk Foremen through the app

Quick call or in-person: show them how to download the app, sign in, and submit a request for themselves or a crew member.

Day 4-7 — Field Worker Onboarding

8

Foremen introduce to their crews

Foremen tell their workers: "Download CleanSpace Stay, sign in with your phone number." That's it. No IT involvement needed.

9

Workers complete their profiles

Workers add their photo, driver's license, loyalty numbers, and TSA info. This lets ops book using their preferences automatically.

Day 5-7 — Accounting Onboarding

10

Walk accounting through reconciliation

Show them the Reconciliation page: how charges are matched to bookings, how to approve/reject matches, and how project tags flow back to QB.

Start with one project. Pick one active project with a known Foreman. Get it running end-to-end (request → approve → book → reconcile) before rolling out to the full company.

3. IT Setup — Email Integration

The platform automatically parses booking confirmation emails (hotels, flights, rental cars) to create booking records. This eliminates manual data entry. There are two options:

Option A: Microsoft Graph (Recommended)
Option B: Auto-Forward Rule

Microsoft Graph — Direct Mailbox Access

This gives the platform direct, real-time access to the travel@acmservicesus.com mailbox. No forwarding rules, no delays, no missed emails.

What IT Needs to Do

  1. Register an App in Microsoft Entra ID (Azure AD)
    • Go to entra.microsoft.com → App registrations → New registration
    • Name: CleanSpace Stay - Mail Reader
    • Supported account types: Single tenant (this organization only)
    • Redirect URI: leave blank (not needed for app-only access)
  2. Add API Permissions
    • API permissions → Add a permission → Microsoft Graph → Application permissions
    • Add: Mail.Read (Read mail in all mailboxes)
    • Click "Grant admin consent for [org name]"
  3. Restrict to the travel@ mailbox only (security best practice)
    • Create a mail-enabled security group containing only travel@acmservicesus.com
    • Run PowerShell (Exchange Online):
    # Connect to Exchange Online Connect-ExchangeOnline # Create application access policy restricting to the security group New-ApplicationAccessPolicy ` -AppId "[App Client ID from step 1]" ` -PolicyScopeGroupId "[Security Group email]" ` -AccessRight RestrictAccess ` -Description "CleanSpace Stay - travel mailbox only"
  4. Create a Client Secret
    • Certificates & secrets → New client secret
    • Description: CleanSpace Stay production
    • Expiry: 24 months (set a calendar reminder to rotate)
  5. Provide these 3 values to CleanSpace Stay admin:
ValueWhere to Find ItExample Format
Tenant IDApp registration → Overview → Directory (tenant) IDa1b2c3d4-e5f6-7890-abcd-ef1234567890
Client IDApp registration → Overview → Application (client) ID12345678-abcd-ef12-3456-7890abcdef12
Client SecretCertificates & secrets → Value (copy immediately, shown only once)abc~DEF123ghiJKL456...
Security note: With the access policy restriction (step 3), the app can ONLY read the travel@ mailbox — not any other mailbox in the organization. This is the Microsoft-recommended approach for least-privilege access.

What the Platform Does With It

  • Polls the travel@ inbox every 5 minutes for new booking confirmation emails
  • Parses hotel, flight, and rental car confirmations automatically
  • Creates booking records in the platform with all details (confirmation #, dates, amounts)
  • Marks processed emails as read (does not delete or move them)
  • Read-only access — the platform cannot send emails or modify the mailbox

Auto-Forward Rule — Simpler but Less Reliable

If Microsoft Graph setup is not feasible, the fallback is an auto-forward rule on the travel@ mailbox. This is simpler for IT but has limitations.

What IT Needs to Do

  1. Open Exchange Admin Center or Outlook Web for the travel@ account
  2. Create a mail flow rule (transport rule):
    • Condition: Messages received by travel@acmservicesus.com
    • Condition: Subject or body contains keywords: confirmation, itinerary, reservation, booking
    • Action: Redirect the message to booking@cleanspacestay.com
  3. Alternatively — forward ALL emails (simpler, the platform ignores non-booking emails):
# Exchange Online PowerShell — forward all travel@ mail Set-Mailbox -Identity "travel@acmservicesus.com" ` -ForwardingSmtpAddress "booking@cleanspacestay.com" ` -DeliverToMailboxAndForward $true
DeliverToMailboxAndForward $true means the original stays in the travel@ inbox AND a copy goes to our platform. Nothing is lost.

Limitations vs. Microsoft Graph

AspectMicrosoft GraphAuto-Forward
ReliabilityDirect API access, never missesDepends on Exchange rule execution
Latency~5 min polling~1-5 min (Exchange processing)
AttachmentsFull access to all attachmentsMay strip some attachments
MaintenanceRotate client secret every 24 monthsRule can break silently on mailbox changes
IT effort30-minute one-time setup5-minute one-time setup

Forwarding Address

Forward to: booking@cleanspacestay.com

This address is owned and monitored by CleanSpace Stay. All emails are processed within seconds of receipt.

4. Accounting Setup — QuickBooks Integration

The platform connects to QuickBooks Online to automatically tag travel expenses to the correct project. This eliminates manual categorization of credit card charges.

How It Works

  1. Credit card charges appear in QuickBooks (from your bank feed as normal)
  2. The platform pulls those charges and matches them to bookings it already knows about
  3. When matched, the platform writes the project tag back onto the QB charge (sparse update — only changes the tag, preserves everything else)
  4. Accounting reviews matches and approves/rejects in the platform
QuickBooks remains the system of record. The platform is a tagging + reconciliation layer on top of it. It never creates, deletes, or modifies the charge amounts — only adds the project dimension.

Connection Steps

1

Connect QB from the platform

Settings → Integrations → Connect QuickBooks Online. You'll be redirected to Intuit to authorize. Sign in with the QB admin account.

2

Choose tagging mode

The platform supports two ways to tag charges. Choose based on how your QB is set up:

Tagging ModeQB Field UsedBest For
Project (Customer/Sub-customer)CustomerRef on each expense lineCompanies using QB Projects or Sub-customers to track jobs
ClassClassRef on each expense lineCompanies using Classes for job costing
3

Map projects

For each CleanSpace project, select the corresponding QB Customer/Class. The platform pulls your QB customer list automatically.

Questions for Accounting / IT

To configure the integration correctly, we need answers to these questions:

#QuestionWhy We Need It
1Do you use QB Projects (sub-customers) or Classes to track jobs?Determines which tagging mode to use
2Do you also use Departments (called "Location" in QB) for any dimension?Platform can write DepartmentRef alongside the project tag
3Which credit card account(s) in QB are used for travel expenses?Platform only pulls charges from these accounts to avoid noise
4Is there a specific expense category (Account) for travel? (e.g., "Travel Expense", "Lodging", "Transportation")Helps the matching algorithm identify travel charges vs. other charges
5Do you want the platform to write a Memo on tagged charges? (e.g., "Tagged by CleanSpace Stay — Booking #1234")Creates an audit trail visible in QB
6Who should have QB admin access to authorize the connection?The OAuth consent requires a QB admin to approve
7Are there any existing automation rules in QB that auto-categorize travel charges?We need to ensure they don't conflict with platform tagging
8Do you want auto-tagging (platform tags high-confidence matches automatically) or review-first (all matches require human approval)?Determines the reconciliation workflow

What Gets Written to QuickBooks

When the platform tags a charge, it writes a sparse update containing only:

{ "sparse": true, "Id": "[existing charge ID]", "SyncToken": "[current sync token]", "DepartmentRef": { "value": "[Location/Dept ID]" }, "Line": [{ "Amount": [unchanged], "AccountBasedExpenseLineDetail": { "AccountRef": [unchanged], "CustomerRef": { "value": "[Project ID]" } } }] }

This means: the charge amount, account, vendor, date, and all other fields remain exactly as they were. Only the project dimension is added.

Safe and reversible. The platform can also un-tag a charge (remove the project dimension) if a match is rejected. No data is ever permanently altered.
